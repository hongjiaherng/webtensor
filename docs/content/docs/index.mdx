**webtensor** is a multi-backend tensor library built for the browser. Train, visualize, and run inference with WebGPU acceleration as a TypeScript library. Backends: CPU (TypeScript), WASM (Rust via wasm-pack), and WebGPU (WGSL compute shaders). All three share full kernel parity across 19 ops.

> **Status: active development — foundation done.** Broadcast-aware autograd, PyTorch-style `matmul` (incl. batched), `sum`/`mean`/`softmax`, JAX-style `compile()` + `grad()`, host-side `SGD`, end-to-end XOR MLP trains on all three backends. See [Start here](/docs/onboarding/start-here) for the full status matrix.

## Architecture at a glance

```text
user code  →  @webtensor/core  (Tensor, ops, autograd, compileGraph)
                    ↓
           @webtensor/runtime  (Engine, Backend interface, view ops)
                    ↓
   backend-cpu | backend-wasm | backend-webgpu
```

The same `Graph` IR runs on any backend. Backends register themselves at import time; `Engine.create('cpu' | 'wasm' | 'webgpu')` picks one.

See [Architecture](/docs/advanced/architecture) for the full contract and [IR reference](/docs/advanced/ir-reference) for the graph schema.
