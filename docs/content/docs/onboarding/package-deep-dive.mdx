Webtensor is a Bun workspace with six packages. The dependency graph is strictly layered: `backend-* ‚Üí runtime ‚Üí ir`, and `core ‚Üí runtime ‚Üí ir`. Backends never depend on `core`; `core` never depends on a specific backend.

![Package dependencies](/diagrams/package-dependencies.svg)

For the boundary contract see [Architecture](/docs/advanced/architecture). This file is the field guide: what types live where, what the public surface is, what to avoid.

---

## `packages/ir`

**Role:** pure data types. Knows nothing about devices, tensors, backends, gradients, or compute.

**Key files:**

- [src/types.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/ir/src/types.ts) ‚Äî `DType`, `AttributeValue`, `Value`, `Node`, `Graph`.
- [src/shape.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/ir/src/shape.ts) ‚Äî `computeContiguousStrides(shape)` (single source of truth).

**Core types** (all from `types.ts`):

| Type             | Purpose                                                                                                 |
| ---------------- | ------------------------------------------------------------------------------------------------------- |
| `DType`          | `'float32' \| 'int32' \| 'bool'`. Re-exported by every other package.                                   |
| `Value`          | One typed edge in the graph: `{ name, shape, dtype, data?, producer?, consumers? }`.                    |
| `Node`           | One operation: `{ id, op, inputs, outputs, attributes? }`. `op` is a string (e.g. `'Add'`, `'MatMul'`). |
| `Graph`          | `{ nodes, values, inputs, outputs, initializers }`. Serializable.                                       |
| `AttributeValue` | Op-specific config ‚Äî numbers, arrays, `ArrayBuffer` for embedded data.                                  |

**Public exports:** `VERSION`, all types above, `computeContiguousStrides`. See [src/index.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/ir/src/index.ts).

**Must not depend on:** anything. This is the leaf. Any backend or runtime utility that creeps in here is a layering violation.

## `packages/core`

**Role:** the user-facing API. Eager `Tensor` class, all op functions, autograd, `compileGraph()`.

**Key files:**

- [src/tensor.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/core/src/tensor.ts) ‚Äî `Tensor` class and `backward()`.
- [src/ops.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/core/src/ops.ts) ‚Äî every op function and its backward closure.
- [src/tensor_init.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/core/src/tensor_init.ts) ‚Äî `tensor()`, `zeros()`, `ones()`.
- [src/compiler.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/core/src/compiler.ts) ‚Äî `compileGraph(outputs)`.
- [src/shape.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/core/src/shape.ts) ‚Äî `broadcastShapes`, `shapeSize`.
- [src/types.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/core/src/types.ts) ‚Äî `OpContext<T>`, `Device`.

**The `Tensor` class** ([tensor.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/core/src/tensor.ts)) carries:

| Field          | What it is                                                                                   |
| -------------- | -------------------------------------------------------------------------------------------- |
| `id`           | Stable string like `t_42` from a module-level counter.                                       |
| `shape`        | `(number \| null)[]` ‚Äî `null` reserved for dynamic dims (placeholders, not yet wired).       |
| `strides`      | C-order by default, computed once at construction.                                           |
| `size`         | Total element count.                                                                         |
| `dtype`        | From `@webtensor/ir`.                                                                        |
| `device`       | `'cpu' \| 'wasm' \| 'webgpu'` ‚Äî picked by the user, propagates through ops.                  |
| `requiresGrad` | Triggers backward closure capture and gradient accumulation.                                 |
| `grad?`        | Set after `.backward()` ‚Äî itself a `Tensor` representing an unevaluated gradient graph.      |
| `_ctx?`        | The op context: `{ op, inputs, attributes?, backward? }`. Captures the closure for autograd. |

The class also exposes chaining methods (`.add()`, `.mul()`, `.transpose()`, ‚Ä¶) that just call into the same op functions in `ops.ts`.

**Must not depend on:** any specific backend. The eager API knows about `Device` strings only ‚Äî the actual backend is selected at `Engine.create(device)` time and registered by side-effect import.

## `packages/runtime`

**Role:** owns execution. Defines the `Backend` contract, exposes `Engine`, dispatches view ops, manages tensor lifetime.

**Key files:**

- [src/engine.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/runtime/src/engine.ts) ‚Äî `Engine`, `registerBackend`, topo sort, refcount GC.
- [src/backend.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/runtime/src/backend.ts) ‚Äî `Backend` interface, `RuntimeTensor`, `RuntimeStorage`, stride utilities (`stridedIdx`, `broadcastStridesOf`, `isContiguous`, `getShapeSize`).
- [src/dtype.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/runtime/src/dtype.ts) ‚Äî `bytesPerElement`, `typedArrayCtor`. All backends import from here.
- [src/views/](https://github.com/hongjiaherng/webtensor/blob/main/packages/runtime/src/views/) ‚Äî `viewRegistry` plus one file per view op (`transpose`, `slice`, `permute`, `expand`, `squeeze`, `unsqueeze`).

**The `Backend` interface** ([backend.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/runtime/src/backend.ts)):

```ts
interface Backend {
  allocate(shape: (number | null)[], dtype: DType): RuntimeTensor;
  read(tensor: RuntimeTensor): Promise<ArrayBufferView>;
  write(tensor: RuntimeTensor, data: ArrayBufferView): void;
  execute(node: Node, inputs: RuntimeTensor[], outputs: RuntimeTensor[]): void | Promise<void>;
  dispose(tensor: RuntimeTensor): void;
}
```

That's it ‚Äî five methods. `dispose()` must be a no-op when `tensor.isView` is true.

**The `Engine` class** holds:

- A `registry` (`Map<string, RuntimeTensor>`) ‚Äî every value the engine has materialized.
- A reference to the chosen `Backend`.
- The `device` string (informational).

`Engine.create(device)` is async and looks up the backend from `backendRegistry`. Backends register themselves at module load time ‚Äî that's why `import '@webtensor/backend-cpu'` is imported only for its side effect.

**Must not depend on:** `core`. The runtime can compute on a `Graph` without knowing how it was built. This separation is what enables future ONNX import ‚Äî drop in a `Graph` from any source, evaluate it.

## `packages/backend-cpu`

**Role:** TypeScript reference backend. The correctness oracle for the other two.

**Key files:**

- [src/backend.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-cpu/src/backend.ts) ‚Äî `CPUBackend` class (~30 lines).
- [src/kernels/registry.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-cpu/src/kernels/registry.ts) ‚Äî kernel map.
- [src/kernels/utils.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-cpu/src/kernels/utils.ts) ‚Äî `CPUKernel` type, helpers like `buf()`.
- [src/kernels/binary/](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-cpu/src/kernels/binary/), `unary/`, `linalg/`, `memory/` ‚Äî one file per op.
- [src/index.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-cpu/src/index.ts) ‚Äî `registerBackend('cpu', async () => new CPUBackend())`.

`CPUKernel` is `(node, inputs, outputs) => void`. Each kernel pulls the typed array via `buf()`, computes broadcast strides if needed, and iterates with `stridedIdx()`. See [Backends and kernels](/docs/onboarding/backends-and-kernels) for kernel anatomy.

**Must not depend on:** `core`.

## `packages/backend-wasm`

**Role:** Rust kernels compiled to WebAssembly via `wasm-pack`. The compiled `.wasm` binary is base64-inlined into the published JS bundle so consumers need no special bundler config.

**Key files:**

- [src/backend.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-wasm/src/backend.ts) ‚Äî `WASMBackend` (async `create()`).
- [src/module.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-wasm/src/module.ts) ‚Äî manual `WebAssembly.instantiate(bytes, importObject)` from a base64-decoded binary; cached via closure. The only WASM import is `__wbindgen_init_externref_table` (provided as a no-op since kernels use only numeric pointers).
- [src/env.d.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-wasm/src/env.d.ts) ‚Äî ambient declaration `declare module '*.wasm' { const base64: string; export default base64; }` so TypeScript matches the bundler's transformation.
- [build.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-wasm/build.ts) ‚Äî Bun build script with a `wasm-inline` plugin that intercepts `*.wasm` imports and rewrites them to `export default "<base64>";`.
- [src/kernels/registry.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-wasm/src/kernels/registry.ts) ‚Äî kernel map.
- [src/kernels/utils.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-wasm/src/kernels/utils.ts) ‚Äî meta-buffer builders (`buildBinaryMetaData` 28 u32, `buildUnaryMetaData` 19 u32, matmul 9 u32).
- [rust/src/](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-wasm/rust/src/) ‚Äî Rust source: `memory.rs` (alloc/free), `utils.rs` (`strided_idx`), `ops/` per kernel.
- `pkg/` ‚Äî generated by `wasm-pack` at build time. **Not published** (only `dist/` ships); consumed by the build plugin and discarded.

**Build:** `bun run --cwd packages/backend-wasm build` ‚Äî runs `wasm-pack build rust --target bundler --out-dir ../pkg` ‚Üí `bun build.ts` (bundles + inlines WASM as base64) ‚Üí `tsc --emitDeclarationOnly`.

**Rust kernel style:** Naive scalar loops; sole crate dependency is `wasm-bindgen 0.2.89` (no `rayon`, no SIMD libs). Memory uses a `Vec` leak/reclaim handoff to JS ‚Äî `alloc_f32` does `Vec::with_capacity(len)` + `mem::forget`, `free_f32` does `Vec::from_raw_parts` to drop. Binary/unary kernels are `for i in 0..total { strided_idx; apply; }` loops. `relu_grad_raw` is a special contiguous-only kernel (no meta buffer) since the autograd engine pre-allocates fresh contiguous tensors. Matmul is naive 3-nested O(MNK), 2D only ‚Äî no tiling, no blocking.

**Must not depend on:** `core`. May depend on `runtime` (for stride utils and `RuntimeTensor`).

## `packages/backend-webgpu`

**Role:** WGSL compute shaders dispatched via the WebGPU API.

**Key files:**

- [src/backend.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-webgpu/src/backend.ts) ‚Äî `WebGPUBackend` (async `create()`, pipeline cache).
- [src/kernels/utils.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-webgpu/src/kernels/utils.ts) ‚Äî `TensorMeta` packing (80-byte / 20-u32 uniform), `packMeta()`, `createMetaBuffer()`, `WebGPUKernel` interface.
- [src/kernels/registry.ts](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-webgpu/src/kernels/registry.ts) ‚Äî kernel map.
- [src/kernels/](https://github.com/hongjiaherng/webtensor/blob/main/packages/backend-webgpu/src/kernels/) ‚Äî one `.wgsl` shader plus `.ts` wrapper per op.

**Critical pitfall:** `meta` is a reserved WGSL keyword. Always name the uniform `u_meta` (or `u_meta_a` / `u_meta_b` for binary ops). Using `meta` causes silent pipeline failure with all-zero output. See [.claude/CLAUDE.md](https://github.com/hongjiaherng/webtensor/blob/main/.claude/CLAUDE.md).

**Must not depend on:** `core`. May depend on `runtime`.

## What lives in `tests/`

- [tests/ops/](https://github.com/hongjiaherng/webtensor/blob/main/tests/ops/) ‚Äî per-op tests (one file per op).
- [tests/backend/](https://github.com/hongjiaherng/webtensor/blob/main/tests/backend/) ‚Äî cross-backend parity (`consistency.test.ts`) and memory/GC tests.
- [tests/foundation/](https://github.com/hongjiaherng/webtensor/blob/main/tests/foundation/) ‚Äî dtype round-trip, view ops, engine core.
- [tests/graph/](https://github.com/hongjiaherng/webtensor/blob/main/tests/graph/) ‚Äî autograd, simple graph execution.
- [tests/helpers.ts](https://github.com/hongjiaherng/webtensor/blob/main/tests/helpers.ts) ‚Äî shared helpers (`BACKENDS`, `runUnary`, `runBinary`, `expectClose`, etc.).

Tests run in **browser mode** via Vitest + Playwright + Chromium. Use `bun run test`, **not** `bun test` ‚Äî the latter bypasses the browser config and WebGPU tests fail. See [Contributor guide](/docs/onboarding/contributor-guide).

## Public API surface

For end users, the entry point is `@webtensor/core`. A typical app imports:

```ts
import { tensor, add, mul, /* ...other ops... */, compileGraph, Engine } from '@webtensor/core';
import '@webtensor/backend-cpu';     // pick one backend (or several)
import '@webtensor/backend-wasm';
import '@webtensor/backend-webgpu';
```

The backend imports are **side-effect imports** ‚Äî they call `registerBackend(device, factory)` at module load. After that, `Engine.create('cpu' | 'wasm' | 'webgpu')` works.

## Layering rules to enforce in code review

| Layer       | May import                             | May NOT import                  |
| ----------- | -------------------------------------- | ------------------------------- |
| `ir`        | (nothing in this repo)                 | anything                        |
| `runtime`   | `ir`                                   | `core`, any `backend-*`         |
| `core`      | `ir`, `runtime`                        | any `backend-*`                 |
| `backend-*` | `ir`, `runtime`                        | `core`, any other `backend-*`   |
| user code   | `core`, `backend-*` (for registration) | `ir`, `runtime` (rarely needed) |

When reviewing a PR, the easiest catch is: did `core` start importing from a `backend-*` package? That's always wrong.

Continue with [Backends and kernels](/docs/onboarding/backends-and-kernels).
