## Where to act

If you want to **unblock real model training** (items 1–5 from the original critical path are done — broadcast-aware autograd, batched matmul, reductions, softmax, SGD, compile-once loop all landed):

1. **Int32 / bool kernels** — unblocks embedding lookups (`gather` with int32 indices), masking, boolean ops. Biggest remaining op-coverage gap.
2. **Loss modules + Adam** — add `crossEntropy` (needs `logSoftmax`), `bceLoss`, and an `Adam` optimizer next to `SGD`. All host-side work.
3. **`concat`, `gather`, slice-backward** — the last few ops needed before ONNX import becomes feasible.
4. **WebGPU `TensorMeta` redesign (rank > 8)** — storage-buffer instead of uniform; removes the rank-8 ceiling.
5. **Dropout + train/eval mode** — tiny addition; unblocks deeper models.

If you want to **harden distribution**:

1. Align package versions (lockstep at `0.1.0` or set up a release tool).
2. Fix the remaining ESLint error.
3. Add CI that runs `bun run test` in browser mode.
4. Add an `examples/` quickstart that uses the published packages (not workspace aliases) — that's the only way to validate the published artifacts.

See [Rearchitect notes](/docs/onboarding/rearchitect-notes) for what each of those touches across the codebase.
