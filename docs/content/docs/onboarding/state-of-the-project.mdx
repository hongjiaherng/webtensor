## Ranks and shapes

| Area                           | Works                                                                                                                                                            | Partial | Missing                                                         |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------- | --------------------------------------------------------------- |
| Strided model                  | All kernels handle arbitrary strides + offset + broadcasting                                                                                                     |         |                                                                 |
| Rank ≤ 64 on all backends      | `MAX_RANK = 64` shared via `@webtensor/ir`; WebGPU `TensorMeta` packs 64 shape + 64 stride slots (520-byte uniform); matmul caps batch rank at 62 to stay within |         |                                                                 |
| Max-rank guard at `allocate()` | Enforced — all backends throw when allocating a tensor with rank > `MAX_RANK`                                                                                    |         |                                                                 |
| Dynamic shapes (`null` dim)    | Type system supports it; placeholder accepts concrete shapes today                                                                                               |         | Shape polymorphism on `compile()` (re-specialize on first call) |

## Where to act

If you want to **unblock real model training** (items 1–5 from the original critical path are done — broadcast-aware autograd, batched matmul, reductions, softmax, SGD, compile-once loop all landed):

1. **Int32 / bool kernels** — unblocks embedding lookups (`gather` with int32 indices), masking, boolean ops. Biggest remaining op-coverage gap.
2. **Loss modules + Adam** — add `crossEntropy` (needs `logSoftmax`), `bceLoss`, and an `Adam` optimizer next to `SGD`. All host-side work.
3. **`gather`, advanced indexing** — remaining ops needed before ONNX import becomes feasible (concat, pad, slice-backward already shipped in Phase 5).
4. **Dropout + train/eval mode** — tiny addition; unblocks deeper models.

If you want to **harden distribution**:

1. Align package versions (lockstep at `0.1.0` or set up a release tool).
2. Fix the remaining ESLint error.
3. Add CI that runs `bun run test` in browser mode.
4. Add an `examples/` quickstart that uses the published packages (not workspace aliases) — that's the only way to validate the published artifacts.

See [Rearchitect notes](/docs/history/rearchitect-notes) for what each of those touches across the codebase.
