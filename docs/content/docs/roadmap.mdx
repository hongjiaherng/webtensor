## Priority: Next Three

### 1. Dtype Kernels (int32 + bool)

**What it is:** Extend op kernels beyond `float32` to support `int32` (and `bool`). The dtype infrastructure is already in place ‚Äî `DType`, `bytesPerElement`, `typedArrayCtor`, dtype-aware `buf()`, and the `dtypeSuffix()` scaffolding in WASM utils. What's missing is per-dtype kernel implementations.

**Why it matters:** `int32` is needed for index ops (Gather, Scatter, embedding lookups ‚Äî required for NLP / classification). `bool` is needed for masks and logical ops.

**What needs to change:**

- **CPU kernels** ‚Äî each existing op branches on `inputs[0].dtype`. Because `buf()` already returns a `TypedArray` union, most kernels just need to substitute an accumulator type and rename `Float32Array` ‚Üí `typedArrayCtor(dtype)(...)` for temp buffers. Comparison ops (`Eq`, `Lt`, `Gt`, `Ne`) produce `bool` outputs.
- **WASM kernels** ‚Äî add per-dtype Rust functions (`add_i32_strided`, `add_u8_strided`, etc.). The existing `dtypeSuffix(dtype)` helper in `src/kernels/utils.ts` already drives function-name lookup ‚Äî add the matching `_i32_` / `_u8_` entries on `WebtensorWasmModule` and wire them up. Meta layouts are already dtype-independent.
- **WebGPU kernels** ‚Äî WGSL has no generics. Ship one `.wgsl` per dtype per op; TS wrappers select pipeline by `inputs[0].dtype`.
- **Promotion rules** ‚Äî adopt PyTorch rules: `add(int32, float32) ‚Üí float32`, etc. Define once in a `resultDType(a, b)` helper.

### 2. Loss functions and training modules

**What it is:** A `@webtensor/core/nn` (or similar) module with `mse_loss`, `cross_entropy`, `log_softmax`, plus a `Parameter` convention so weights can be managed uniformly.

**Why it matters:** Today training uses `reduceMean(mul(diff, diff))` ad-hoc. Named losses + `log_softmax` unlock classification training. This also paves the way for an Adam optimizer and richer schedulers.

**What needs to change:**

- **`log_softmax`** under `packages/core/src/ops/activation/` ‚Äî numerically stable: `x - logsumexp(x, axis)`. Backward: `grad - softmax(x) * sum(grad, axis, keepdim=true)`.
- **`mse_loss(pred, target)` = `reduceMean(pow(sub(pred, target), 2))`** ‚Äî one-liner alias.
- **`cross_entropy(logits, target)`** ‚Äî combined `log_softmax` + negative-log-likelihood.
- **`Adam`** ‚Äî another host-side optimizer next to `SGD` in `packages/nn/src/optim/`. Keeps the existing placeholder-fed pattern (weights live as `Float32Array`s, not inside the graph).

---

### 3. Gather + advanced indexing

**What it is:** Fill the remaining gaps in the op set that unblock real models. `concat`, `pad`, and `slice` backward already shipped in Phase 5. Remaining: `gather(data, indices, axis)` and scatter-style advanced indexing.

**Why it matters:** `gather` with int32 indices enables embedding lookups (blocked on int32 kernels). Scatter is needed for advanced indexing and some backwards.

**What needs to change:**

- **`gather`** ‚Äî blocked on int32 kernels (indices must be int32). Kernel reads `data[indices[i]]` per output element. Backward: `scatter_add`.
- **`scatter` / `scatter_add`** ‚Äî write values into an output tensor at indexed positions.
